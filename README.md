## 📋 Описание

YamBassPlayer — это кроссплатформенный музыкальный плеер с текстовым интерфейсом (TUI), который позволяет воспроизводить музыку из Яндекс.Музыки прямо в терминале. Приложение поддерживает визуализацию аудио-спектра, 10-полосный эквалайзер, управление плейлистами, локальный и онлайн-поиск треков, историю прослушивания со статистикой и несколько цветовых тем оформления.

## 📦 Установка

### Требования

- Токен Яндекс.Музыки

### Шаги установки

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/Blesya/YaBassPlayer.git
   cd YaBassPlayer
   ```

2. **Соберите проект:**
   ```bash
   cd YamBassPlayer
   dotnet build
   ```

3. **Запустите приложение:**
   ```bash
   dotnet run --project YamBassPlayer
   ```
4. **Задайте токен**

   При первом запуске откроется диалог ввода токена. Введите токен Яндекс.Музыки и нажмите OK.

## 🎮 Использование

### Управление интерфейсом

- **Навигация:** клавиши `↑` `↓` `←` `→` для перемещения по плиткам треков и плейлистам
- **Выбор:** `Enter` для воспроизведения трека или выбора плейлиста
- **Двойной клик:** воспроизведение трека
- **Прокрутка:** колесо мыши для прокрутки списка треков
- **Меню:** `F9` или `Alt` для открытия меню
- **Выход:** выберите "Файл" → "Выход" в меню или `Ctrl+C`

### Управление воспроизведением

- **Play/Pause:** кнопка в интерфейсе или двойной клик по треку
- **Stop:** кнопка остановки в интерфейсе
- **Prev/Next:** переключение между треками в очереди
- **Seek:** клик по прогресс-бару для перемотки к нужной позиции
- **Избранное (локальное):** кнопка `+` / `-` рядом с таймером
- **Избранное (Яндекс):** кнопка `♡` / `♥` для лайка/дизлайка в Яндекс.Музыке

### Плейлисты

При запуске загружаются плейлисты из Яндекс.Музыки, а также формируются локальные:

- **Мои треки** — треки, которые вам понравились в Яндекс.Музыке
- **Локальное Избранное** — избранные треки, сохранённые локально
- **Загруженные** — треки из локального кэша (в обратном хронологическом порядке)
- **Топ 10** — 10 самых прослушиваемых треков (по истории)
- **Топ вечеров** — топ треков, прослушанных с 16:00 до 24:00
- **Топ по дням** — группа плейлистов с топ-треками по каждому дню недели
- **Персональные плейлисты** — плейлисты из вашего аккаунта Яндекс.Музыки

### Меню приложения

- **Файл** → Выход
- **Темы** → выбор темы оформления
- **Аудио** → Эквалайзер (10-полосный, настройки сохраняются)
- **Инструменты** → Локальный поиск, Поиск по ЯМ, Статистика БД

### Темы оформления

Доступные темы в меню "Темы":
- **Тёмная** — классическая тёмная тема
- **Светлая** — светлая тема с серым фоном
- **Белая** — светлая тема с белым фоном
- **Матрица** — зелёная тема в стиле "Матрицы"
- **Киберпанк** — яркая неоновая тема
- **Спокойная** — мягкая тема в стиле Nord
- **По умолчанию** — стандартная тема Terminal.Gui

Выбранная тема сохраняется и автоматически применяется при следующем запуске.

## 🏗️ Архитектура

Проект построен на паттерне **MVP** (Model-View-Presenter) с внедрением зависимостей через **Autofac**. Все компоненты регистрируются в `ServicesProvider.cs` как синглтоны и разрешаются автоматически.

### Паттерн MVP

```
┌──────────┐     события      ┌─────────────┐     вызовы     ┌──────────┐
│   View   │ ───────────────► │  Presenter  │ ─────────────► │ Services │
│ (TUI)    │ ◄─────────────── │  (логика)   │ ◄───────────── │ (данные) │
└──────────┘   обновление UI  └─────────────┘   результаты   └──────────┘
```

- **View** не содержит бизнес-логики — только отображение и перехват пользовательских действий
- **Presenter** обрабатывает события от View, вызывает сервисы и обновляет View
- **Services** инкапсулируют работу с данными, API и аудио

Каждый компонент имеет интерфейс (`I*`) и реализацию в подпапке `Impl/`.

### Слои приложения

| Слой | Директория | Описание |
|------|-----------|----------|
| **Models** | `Models/`, `Enums/` | Доменные модели: `Track`, `Playlist`, `PlaylistGroup`, `PlaylistTreeItem`, `DatabaseStatistics`, перечисление `PlaylistType` |
| **Views** | `Views/` | 8 представлений: плейлисты, плитки треков, статус воспроизведения, спектр, эквалайзер, поиск (локальный и Яндекс), статистика, визуализация |
| **Presenters** | `Presenters/` | 8 презентеров, управляющих соответствующими View |
| **Services** | `Services/` | 13 сервисов для бизнес-логики |
| **Configuration** | `Configuration/` | Чтение/запись `appsettings.json` (токен, эквалайзер, тема) |

### Ключевые сервисы

| Сервис | Интерфейс | Назначение |
|--------|-----------|------------|
| `AudioPlayerService` | `IAudioPlayer` | Воспроизведение через BASS: play, pause, stop, seek, FFT-данные для спектра |
| `TrackRepository` | `ITrackRepository` | Агрегация треков из разных источников: Яндекс.Музыка, кэш, история, избранное, поиск |
| `TrackFileProvider` | `ITrackFileProvider` | Загрузка и кэширование аудиофайлов в папку `tracks/` |
| `TrackInfoProvider` | `ITrackInfoProvider` | Получение и кэширование метаданных треков в SQLite |
| `PlaybackQueue` | `IPlaybackQueue` | Очередь воспроизведения с навигацией prev/next |
| `HistoryService` | `IHistoryService` | История прослушивания: запись, топ-треки, статистика по времени |
| `ListenTimer` | `IListenTimer` | Таймер прослушивания — фиксирует начало и конец воспроизведения трека |
| `LocalFavoriteService` | `ILocalFavoriteService` | Локальное избранное (SQLite) |
| `YandexFavoriteService` | `IYandexFavoriteService` | Лайки/дизлайки в Яндекс.Музыке |
| `BassEqualizer` | `IBassEqualizer` | 10-полосный эквалайзер, привязка к аудиопотоку |
| `DatabaseProvider` | `IDatabaseProvider` | Единственное SQLite-подключение на приложение |
| `DatabaseStatisticsService` | `IDatabaseStatisticsService` | Статистика: количество треков, прослушиваний, размер кэша |
| `AuthService` | `IAuthService` | Авторизация в Яндекс.Музыке по токену |

### Поток данных при воспроизведении

```
Пользователь выбирает трек в TracksTileView
  → PlaybackQueue устанавливает текущий трек
    → MainWindow получает событие OnTrackChanged
      → TrackInfoProvider загружает метаданные из SQLite / Яндекс API
      → TrackFileProvider скачивает файл (или берёт из кэша tracks/)
        → AudioPlayerService создаёт BASS-поток и начинает воспроизведение
          → BassEqualizer привязывается к потоку
          → SpectrumView получает FFT-данные каждые 16 мс
          → ListenTimer фиксирует время прослушивания
```

За 30 секунд до конца трека `AudioPlayerService` вызывает событие `OnPreloadRequested` — `MainWindow` заранее скачивает следующий трек из очереди.

### Хранение данных

- **SQLite** (`tracks_cache.db`) — метаданные треков (название, исполнитель, альбом), история прослушиваний с временными метками, локальное избранное
- **Файловый кэш** (`tracks/`) — загруженные аудиофайлы в формате MP3, именуются по ID трека
- **JSON-конфигурация** (`appsettings.json`) — токен авторизации, настройки эквалайзера (10 полос), выбранная тема

### Жизненный цикл приложения

1. **Запуск** — `Program.cs` проверяет наличие токена; при первом запуске показывает `TokenInputDialog`
2. **Авторизация** — `AuthService` авторизуется в Яндекс.Музыке по токену из `appsettings.json`
3. **Инициализация DI** — `ServicesProvider.Initialise()` регистрирует все компоненты в Autofac
4. **Инициализация аудио** — `AudioPlayerService.Init()` запускает движок BASS
5. **Запуск TUI** — `MainWindow` строит интерфейс: меню, плейлисты, плитки треков, спектр, панель воспроизведения
6. **Загрузка плейлистов** — `PlaylistsPresenter` загружает плейлисты из Яндекс.Музыки и формирует локальные (топы, избранное, кэш)
7. **Работа** — пользователь выбирает плейлисты и треки, приложение скачивает и воспроизводит музыку
8. **Завершение** — освобождение BASS-ресурсов, закрытие SQLite-подключения
